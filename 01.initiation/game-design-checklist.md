# Чек-лист: Game Design

- [ ]  **Интересные решения** — каждое значимое действие имеет ≥2 живых варианта, без очевидно лучшего
- [ ]  **Агентность** — исход зависит от игрока, а не от системы; нет ощущения «лишнего»
- [ ]  **Поток** — сложность растёт вровень с навыком: не скучно и не невыносимо
- [ ]  **Озарение, а не фрустрация** — проиграл и понял почему; «я не мог знать» = провал
- [ ]  **Баланс выбора** — есть trade-off, нет доминантной стратегии и ложных выборов
- [ ]  **Эмоциональная кривая** — завязка → нарастание → кульминация → развязка, острый финал
- [ ]  **Реиграбельность** — глубина и/или вариативность, партии не повторяются
- [ ]  **Чёткий тип удовольствия** — метит в 1–2 (Challenge, Fellowship, Discovery…), не размазано
- [ ]  **MDA выстроен** — правила порождают нужную динамику и эмоцию
- [ ]  **Ясная точка входа** — механика/тема/опыт: понятно, на чём держится игра
- [ ]  **Честный рандом** — псевдослучайность, ощущаемая справедливой; нет чувства «игра жульничает»
- [ ]  **Глубина без перегруза игрока** — компьютер ведёт всю «бухгалтерию», сложность скрыта под капотом
- [ ]  **Онбординг/туториал** — механики раскрываются прогрессивно, а не вываливаются сразу
- [ ]  **Juice (сок)** — мгновенный аудиовизуальный отклик на каждое действие (анимация, звук, отдача)
- [ ]  **Интерфейс обучает сам** — игрок осваивает через UI, не через чтение правил

**Главный неформальный тест (перекрывает все галочки)**

- [ ]  **«Ещё один заход»** — после сессии хочется сыграть снова
